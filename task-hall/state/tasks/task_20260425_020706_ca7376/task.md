# 任务：设计玄幻小说写作 agent 工作流

## 类型

custom

## 背景

用户希望在 agent 工作区中建立玄幻小说写作相关能力。当前 workspace 以 `D:\\agent_workspace\\capability-library` 为核心，`mycli` / `agent-cli` / `task-hall` 已支持任务发布、领取、报告与定时唤起。需要一套清晰的、可复用的多 agent 协作工作流设计，后续可用于实现 `mycli agent-workflow`、skill、任务大厅拆分任务或手工协作。

玄幻小说方向需要重点考虑长篇连载特性，包括但不限于：设定一致性、升级体系、爽点节奏、伏笔回收、人物弧光、章节钩子、分卷结构以及审稿反馈循环。

## 目标

设计一套可由多个 agent 协作执行的玄幻小说写作 workflow 方案，覆盖从题材定位、世界观设计、人物设计、主线大纲、分卷节奏、章节细纲，到正文产出与质量复审的完整流程。

重点在于输出可复用、可落地、可拆分给 agent 的流程方案，而不是直接创作一部长篇小说正文。

## 任务说明

请产出一份面向多 agent 协作的玄幻小说写作工作流方案文档，至少应覆盖以下内容：

1. 工作流目标与适用范围。
2. agent 角色划分与职责边界。
3. 阶段流程图或等价的步骤清单。
4. 每阶段的输入、输出、质量门槛。
5. 可直接复用的关键提示词模板。
6. 面向 task-hall 的任务拆分建议。
7. 可执行的验收清单。
8. 后续落地到 `mycli agent-workflow` 或 `skill-library` 的建议。

方案中应明确：

- 哪些步骤适合并行执行。
- 哪些步骤必须串行推进。
- 长篇一致性资产如何建立与维护。
- 正文生成前、中、后的质量控制机制。
- 复审与返工闭环如何运作。

## 约束

- 不要直接创作完整长篇小说正文，重点交付可复用工作流设计。
- 方案必须适合 agent 协作，明确哪些步骤可并行、哪些步骤必须串行。
- 必须考虑长篇一致性维护，例如世界观 bible、角色卡、伏笔表、时间线、术语表。
- 必须包含质量控制和复审环节，避免只给抽象建议。
- 不得要求联网或外部付费服务作为必要条件。
- 如建议创建后续任务，应写清每个任务的目标、交付物和领取门槛。
- 输出文档应写入明确保存路径；建议路径为 `D:\\agent_workspace\\tmp\\xuanhuan-novel-agent-workflow.md`，也可由执行者选择其他明确路径。

## 交付物

一份 Markdown 方案文档，建议保存到：

- `D:\\agent_workspace\\tmp\\xuanhuan-novel-agent-workflow.md`

文档至少包含：

1. 工作流目标与适用范围。
2. agent 角色划分。
3. 阶段流程图或步骤清单。
4. 每阶段输入、输出、质量门槛。
5. 关键提示词模板。
6. 任务大厅拆分建议。
7. 验收清单。
8. 后续落地到 `mycli agent-workflow` 或 `skill-library` 的建议。

## 验收标准

- 文档为 Markdown 格式，结构完整、可直接阅读和复用。
- 明确给出从题材定位到正文产出与复审的端到端流程。
- 明确标注并行步骤与串行步骤。
- 明确给出一致性资产清单及维护机制，至少覆盖世界观 bible、角色卡、伏笔表、时间线、术语表。
- 明确给出质量控制与复审流程，包括返工触发条件或检查点。
- 明确给出 task-hall 拆分建议，且每个后续任务都包含目标、交付物和领取门槛。
- 不将联网能力或外部付费服务设为必要前提。
- 给出与 `mycli agent-workflow` 或 `skill-library` 集成的落地建议，而非仅停留在抽象原则层面。
