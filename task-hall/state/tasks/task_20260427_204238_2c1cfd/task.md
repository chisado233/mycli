# 五万字玄幻小说创作任务

## 背景

用户希望通过任务大厅发布一个创作任务：写一本约五万字的玄幻小说。该任务属于长篇创作任务，不是代码实现任务。执行 agent 需要能够进行世界观设定、剧情规划、章节拆分、持续写作和最终整理。

## 目标

创作一部完整的中文玄幻小说，正文总字数约 50,000 字。

## 任务说明

请完成一部结构完整、可读性强、风格统一的玄幻小说。小说需要至少包含：

- 书名
- 简介
- 世界观设定
- 主要人物设定
- 章节大纲
- 完整正文
- 结尾收束

建议先完成总体规划，再按章节持续写作。正文应避免只给大纲或片段，最终交付必须是可阅读的完整小说。

## 推荐流程

1. 确定题材基调、核心卖点和目标读者。
2. 设计世界观、修炼体系、主要势力、核心冲突。
3. 设计主角、重要配角、反派和人物成长线。
4. 拆分章节大纲，建议 15-25 章左右。
5. 分批写作正文，保持情节连续、设定一致。
6. 完成全文后做一次整体自检：
   - 字数是否接近 50,000 字；
   - 是否有明显断章、跳章、重复段落；
   - 人名、地名、境界名是否一致；
   - 主线是否完成闭环；
   - 结尾是否收束。
7. 整理最终 Markdown 文档并提交任务报告。

## 约束

- 必须原创，不要改写已有知名作品。
- 不要抄袭现有小说、影视、游戏或动漫剧情。
- 内容应适合普通中文读者阅读，避免露骨色情、极端血腥、现实仇恨煽动等不适内容。
- 可以使用常见玄幻类型元素，但需要有自己的设定组合和人物动机。
- 如果因单次上下文长度限制无法一次完成 50,000 字，应明确分卷/分批写作，并持续产出到完整交付为止。
- 最终交付物应保存到 `D:\agent_workspace\projects\xuanhuan-novel-workflow` 或其子目录；如果需要新建项目目录，应保持目录结构清晰。

## 交付物

- 一份完整小说 Markdown 文件，建议命名类似：`novel.md`。
- 一份创作说明或 README，说明：
  - 书名；
  - 总字数估算；
  - 章节列表；
  - 世界观与主线概述；
  - 产物路径。
- 如分批写作，可保留章节草稿目录，但最终必须有合并后的完整正文。
- 任务完成后提交结构化任务报告，说明完成内容、产物路径、字数估算和自检结果。

## 验收标准

- 小说正文总字数约 50,000 字，允许合理浮动，但不应只写几千字或只交大纲。
- 有明确开端、发展、高潮和结局。
- 主角成长线完整，核心冲突有解决。
- 世界观、境界体系、人物关系在全文中基本一致。
- 最终 Markdown 文件可直接阅读。
- 任务报告包含产物路径、字数估算、自检结果和未完成项说明；若已完成，应明确没有关键未完成项。

## 领取门槛

复杂度：高

最低模型要求：strong

推荐 agent 类型：engineering-leader

领取要求说明：

本任务是长篇创作与组织任务，建议由 leader 领取后自行判断是否拆分给 writing/builder 类型 agent。领取 agent 必须能进行长上下文规划、分阶段交付和最终整合，不建议 nano/cheap 模型领取。

## 备注

这是创作任务，不需要执行代码构建。若系统中没有专门 writing agent，可以由 engineering-leader 组织完成或发布子任务进行分章节写作。
