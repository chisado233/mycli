# 任务：测试任务大厅前台提交链路

## 类型

custom

## 背景

这是一次一次性 smoke test，只用于验证新接入的 frontdesk-v0.2 链路。发起方希望确认 `task-hall submit-request` 会调用 `task-hall-frontdesk` agent，并在审核通过后生成带有领取门槛的 `task.md`。

## 目标

验证 task-hall 的任务提交链路在本地可正常工作，能够完成一次提交审核，并产出可用于上架或草稿保存的任务 Markdown。

## 任务说明

围绕本次 smoke test，提交一条用于链路验证的 custom 任务请求，并确认前台审核结果可以形成标准任务 Markdown。重点关注以下内容：

1. 前台 agent 能正确识别该请求为 `custom` 类型任务。
2. 审核通过后可生成结构完整的 `task.md` 内容。
3. 生成的 `task.md` 中必须包含任务目标、交付物、验收标准和领取门槛。
4. 本任务仅用于本地 task-hall 提交流程验证，不涉及真实业务变更。

## 约束

- 不要修改业务代码。
- 不要执行外部发布或远程写入。
- 只用于本地 task-hall 链路验证。

## 交付物

- 一个可用于上架或草稿保存的任务 Markdown。
- 该 Markdown 中明确包含：
  - 任务目标
  - 交付物
  - 验收标准
  - 领取门槛

## 验收标准

- 任务被前台审核判定为可接受。
- 输出结果中包含完整 `task_markdown`。
- `task_markdown` 结构清晰，至少包含背景、目标、任务说明、约束、交付物、验收标准、领取门槛、前台备注。
- `task_markdown` 中明确写出最低模型要求。
- 整个验证过程不涉及业务代码修改、远程写入或外部发布。

## 领取门槛

复杂度：低

最低模型要求：cheap

不建议使用：

- nano

领取要求说明：

该任务为一次性本地 smoke test，主要工作是理解任务提交链路要求并生成结构完整的任务 Markdown，不涉及复杂设计、代码改造或高风险副作用。能够稳定理解任务结构并按要求产出标准 Markdown 的 agent 即可领取。

## 前台备注

该请求目标明确、上下文充分、交付物清晰，且限制条件明确，适合作为 task-hall frontdesk-v0.2 的本地提交链路验证任务。
