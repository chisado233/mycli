# Chat Soft 开发计划

## 1. 总体阶段

### 阶段 0：方案落地

- 完成正式需求文档
- 完成总体架构设计
- 锁定技术栈
- 购买并准备服务器

### 阶段 1：后端骨架

- 初始化服务端工程
- 建立 PostgreSQL / Redis / Docker Compose
- 完成设备注册与配对
- 完成 WebSocket 基础链路

### 阶段 2：V1 文本消息

- 文本消息发送
- 文本消息接收
- 消息状态流转
- 本地落库
- 服务端补收

### 阶段 3：V1 语音消息

- 安卓录音
- Windows 语音消息收发
- 音频文件上传下载
- 进度播放

### 阶段 4：多设备同步与通知

- 消息同步游标
- 离线补收
- 安卓通知
- Windows 通知

### 阶段 5：电脑端 AI 接口

- 本地 HTTP API
- 本地 WebSocket
- CLI

## 2. V1 验收标准

- 文本消息跨网稳定可用
- 语音消息跨网稳定可用
- 设备离线后可补收消息
- Windows 端 API 可被本地 AI 调用

## 3. 当前建议的开发顺序

1. 服务器与数据库先跑通
2. Windows 客户端先做最小原型
3. 安卓客户端再接入
4. 文本消息优先
5. 语音消息第二
6. AI 接口第三

- 初始化 monorepo
- 搭建 server 原型
- 搭建 desktop 原型
- 搭建 android 原型
