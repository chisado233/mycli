# Chat Soft 产品需求文档

## 1. 产品定位

Chat Soft 是一套面向个人私有使用的跨网络通信系统，核心目标是打通“安卓手机 + Windows 电脑 + 后续多设备”之间的稳定消息通信能力，并为电脑端提供可供 AI 调用的开放接口。

它不是面向公众的通用社交软件，不优先考虑陌生人社交、公开注册、运营后台、泛用户增长等能力。

## 2. 产品目标

### 2.1 总目标

- 让个人多设备之间稳定跨网络通信
- 具备长期保存、同步和补收消息的能力
- 后续逐步扩展到图片、视频、文件、实时音视频
- 电脑端具备 GUI + 本地自动化接口双重能力

### 2.2 V1 目标

- 跑通文本消息
- 跑通语音消息
- 能跨网络使用
- 能保存历史消息
- 能测量实际延迟和稳定性
- 电脑端可通过本地接口发送与读取基础消息

## 3. 用户与场景

### 3.1 目标用户

- 唯一主要用户：项目拥有者本人

### 3.2 核心场景

- 手机给电脑发文本消息
- 电脑给手机发文本消息
- 手机给电脑发语音消息
- 电脑查看历史消息
- 某个设备离线后恢复在线并补收消息
- AI 在电脑端读取最近消息并触发自动发送

## 4. 平台范围

### 4.1 V1 平台

- 安卓客户端
- Windows 客户端
- 香港云服务器

### 4.2 后续平台

- 后续可扩展更多安卓设备
- 后续可扩展更多 Windows 设备

## 5. 非目标

V1 不做：

- 账号注册登录
- 面向公众的好友体系
- 群聊
- 多人音视频
- 图片/视频/文件完整能力
- 复杂权限系统
- Web 端

## 6. 功能需求

### 6.1 V1 必做功能

- 文本消息发送
- 文本消息接收
- 语音消息录制
- 语音消息发送
- 语音消息接收与播放
- 消息状态：
  - 发送中
  - 已发送
  - 已送达
  - 已读
  - 失败
- 本地历史消息存储
- 服务端历史消息存储
- 离线补收
- 多设备同步基础能力
- 安卓通知
- Windows 桌面通知
- 电脑端 AI 接口：
  - 本地 HTTP API
  - 本地 WebSocket
  - 命令行接口

### 6.2 V2 功能

- 图片消息
- 普通音频文件发送
- 普通视频文件发送
- 文件上传下载
- 断点续传
- 搜索历史
- 更完整的设备管理

### 6.3 V3 功能

- 1 对 1 实时语音通话
- 1 对 1 实时视频通话
- 更强弱网重连
- 更完整多设备体验

## 7. 核心交互

### 7.1 文本消息

- 输入文本
- 点击发送
- 本地立即落库并进入发送中状态
- 服务端确认后更新状态
- 目标端收到后落库并通知

### 7.2 语音消息

- 按住开始录音
- 松开发送
- 本地先保存录音文件
- 上传到服务端对象存储或媒体文件区
- 对方收到消息卡片与音频地址
- 对方可播放，显示进度条

## 8. 数据要求

- 本地和服务端都保留消息历史
- 服务端支持离线消息补收
- 语音消息需要单独媒体存储
- 后续应支持大文件传输扩展

## 9. 安全要求

V1 原则：

- 传输链路必须使用 TLS
- 设备与服务端之间使用设备级鉴权
- 服务端不得采用明文弱鉴权
- 本地数据库需预留加密能力
- 电脑端 AI 接口必须限制为本机访问
- AI 接口需要访问令牌或本地授权机制

说明：

V1 不强行要求一开始就完成最重的端到端加密体系，但整体架构需要为后续端到端加密预留空间。

## 10. 成功标准

V1 达标标准：

- 手机和电脑跨网络可稳定互发文本
- 手机和电脑跨网络可稳定互发语音消息
- 单条文本消息端到端延迟满足可接受范围
- 语音消息发送与接收稳定
- 离线重连后可补收消息
- 电脑端 AI 接口能完成基础发送与读取
