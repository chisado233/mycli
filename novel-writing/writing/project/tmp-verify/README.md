# 动态资产库说明

> 目标：用一套持续更新的动态资产库管理小说中的人物、势力及其在不同剧情区间的状态。
>
> 核心模型：资产本体 + 区间状态 + 状态变更日志 + 新增对象收件箱。

## 1. 核心概念

### 资产本体

资产本体回答：作品里有哪些对象。

当前重点资产类型：

- 人物
- 势力

后续可扩展：

- 地域
- 资源/器物
- 信息/秘密
- 伏笔

资产本体记录相对稳定的信息，例如名称、类型、重要度、首次出现位置、是否有独立 md、是否进入状态追踪。

### 区间状态

区间状态回答：在某个剧情区间内，这个人物/势力处于什么状态。

推荐状态区间表达：

```text
第 N 章结束后 / 第 N+1 章开始前
```

例如：

```text
ch005-ch006：第 5 章结束后，第 6 章开始前
ch006-ch007：第 6 章结束后，第 7 章开始前
```

写作时遵循：

```text
读取前置状态 → 写本章 → 输出后置状态
```

也就是：

```text
ch005-ch006 状态 + 第 6 章事件 = ch006-ch007 状态
```

### 状态变更日志

状态变更日志回答：从上一个区间到当前区间发生了什么变化，变化原因是什么，后续影响是什么。

### 新增对象收件箱

新增对象收件箱用于暂存大纲细化或正文写作中临时冒出来的人物/势力，避免它们直接污染全局资产库。

---

## 2. 状态粒度

状态粒度应随着大纲不断细化：

```text
全书级
  ↓
卷级
  ↓
小篇章级
  ↓
五章级
  ↓
章节间
```

### 全书级

记录开局、中期、后期、终局的大状态。

### 卷级

记录每卷开始和结束时的人物、势力、关系、资源、信息状态。

### 小篇章级

记录一个连续事件单元的开始、结束和关键变化。

### 五章级

记录五章组的开始、结束和预计/实际变化。

### 章节间

记录某章写完后、下一章开始前的精确状态。

---

## 3. 计划状态与实际状态

状态必须区分来源：

- 计划状态：由粗纲、卷纲、小篇章纲、五章纲推导。
- 正文确认状态：由正文写作后确认。
- 修订状态：人工或 AI 审核后修改确认。

如果正文实际状态偏离大纲计划，优先以正文实际状态为准，并反向更新后续状态表。

---

## 4. 完整快照与差量变更

不要每章都完整重写所有人物和势力状态。

推荐：

- 关键节点写完整快照：卷开始/卷结束、小篇章开始/结束、五章组开始/结束。
- 每章写差量变更：只记录本章发生变化的人物、势力、关系、资源、信息、伏笔。

当前状态可以由：

```text
最近完整快照 + 后续差量变更
```

合成。

---

## 5. 新增对象规则

大纲细化和正文写作中新增的人物/势力，先进入“新增对象收件箱”。

每个阶段结束时处理收件箱，决定：

- 入库
- 补模板
- 升级
- 合并到已有对象
- 仅局部保留
- 归档/废弃

新增前必须检查：

> 这个新对象能不能由已有对象承担？

如果可以，优先复用已有资产，避免人物和势力膨胀。

---

## 6. 重要度分级

人物和势力都建议分级管理：

| 等级 | 含义 | 处理方式 |
| --- | --- | --- |
| S | 全书核心 | 完整三层模板 + 全程状态追踪 |
| A | 长期重要 | 至少基础信息层 + 剧情功能层 + 关键状态追踪 |
| B | 阶段重要 | 简版模板 + 本卷/小篇章状态追踪 |
| C | 功能对象 | 资产登记 + 必要时局部追踪 |
| D | 一次性对象 | 一般不入库，除非影响状态 |

---

## 7. 写作使用流程

每写一章时：

```text
输入：
1. 当前状态快照
2. 本章细纲
3. 涉及人物 md
4. 涉及势力 md
5. 前文摘要

输出：
1. 章节正文
2. 本章状态变更日志
3. 下一章节间状态
4. 新增对象收件箱更新
```

---

## 8. 硬规则

- 状态表记录“当前状态”，不是完整设定。
- 完整设定看人物/势力 md。
- 状态变化必须有来源，可追溯到大纲或正文。
- 谁知道什么必须单独记录。
- 每章写完必须写状态变更日志；无变化也要记录“无重大变化”。
- 写下一章前必须读取当前状态快照。
- 正文实际状态优先于大纲计划状态。
- 新增人物/势力先入收件箱，不直接污染资产库。
