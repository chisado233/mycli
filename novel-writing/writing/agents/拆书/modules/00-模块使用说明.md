# 拆书模块化提示词使用说明

本目录保存拆书流程中各部分的“注意事项 + 输出模板”。

主拆书 agent 不应把所有细则都写死在一个超长 prompt 中，而应在 request 的 `context_files` 中按需传入这些模块。

## 推荐调用方式

### 批次素材拆解

传入：

- `01-素材库总规则.md`
- `02-场景素材模板.md`
- `03-爽点素材模板.md`
- `04-伏笔素材模板.md`
- `05-人物设定素材模板.md`
- `06-情感线推进模板.md`
- `07-力量体系素材模板.md`
- `08-世界设定素材模板.md`
- `09-完整故事概要模板.md`
- `10-文学风格收集模板.md`
- `11-灵活记录模板.md`
- `12-剧情内容拆解模板.md`
- `15-质量自检模板.md`

### 文风拆解

传入：

- `10-文学风格收集模板.md`
- `13-文风拆解模板.md`
- `15-质量自检模板.md`

### 阶段/全书统一

传入：

- `14-最终统一模板.md`
- 对应分类模板，如人物、情感线、力量体系、世界设定、伏笔。

## 路径原则

这些模块只是提示词上下文，不是素材产物本身。正式产物仍应写入拆书项目的 output 目录，或经人工筛选后写入 material-library。
