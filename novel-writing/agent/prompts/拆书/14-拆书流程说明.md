# 完整拆书 Agent 流程说明

## 目标

本流程用于把一本小说，尤其是几十万到几百万字的长篇小说，拆解成与“小说项目模板”同构的“对标作品拆解项目”。

核心产物不是剧情梗概，而是可迁移到创作项目中的结构、机制、人物、设定、情绪、伏笔与章节技法。

## 一、目录结构

最终拆书项目建议使用：

```text
00-拆解总览/
01-作品企划/
02-世界观/
03-故事背景/
04-角色/
05-大纲/
06-明线暗线伏笔/
06A-状态追踪/
07-章节拆解/
08-审核与评估/
09-素材引用/
10-拆解记录/
11-调度与拆解/
12-长篇上下文管理/
```

## 二、完整流程

```text
1. 拆书总控 agent 制定计划
2. 原文导入切分 agent 整理章节
3. 拆书模板初始化 agent 建立项目结构
4. 章节粗拆 agent 分批处理章节
5. 索引更新 agent 更新人物/伏笔/设定/情绪索引
6. 阶段汇总 agent 生成阶段总结和滚动摘要
7. 卷级汇总 agent 生成卷纲式拆解
8. 章节精拆 agent 精读 A 级重点章节
9. 高光素材摘录 agent 提取搞笑点、抽象点、名场面、名句、爽点原文短摘并解析
10. 角色拆解 agent 生成角色文件和人物功能表
11. 世界观设定拆解 agent 生成设定机制分析
12. 明线暗线伏笔 agent 生成问题链与伏笔路径
13. 作品企划反推 agent 反推卖点、目标受众、情绪承诺
14. 可借鉴点提炼 agent 输出可借鉴点总表
15. 审核评估 agent 检查完整性、证据、可迁移性和风险
```

## 三、长篇上下文策略

不要一次读完整本书。使用：

```text
当前批次章节原文
+ 上一阶段滚动摘要
+ 人物/伏笔/设定/情绪索引
+ 输出模板
```

每一轮处理后都要更新：

- `12-长篇上下文管理/滚动摘要.md`
- `12-长篇上下文管理/人物索引.md`
- `12-长篇上下文管理/伏笔索引.md`
- `12-长篇上下文管理/设定索引.md`
- `12-长篇上下文管理/待验证问题.md`
- `12-长篇上下文管理/名场面索引.md`
- `12-长篇上下文管理/名句索引.md`
- `12-长篇上下文管理/梗与抽象点索引.md`

## 四、建议执行顺序

### 快速拆

适合判断一本书是否值得深入学习。

```text
总控 → 导入切分 → 前30章粗拆 → 阶段汇总 → 作品企划反推 → 可借鉴点提炼 → 审核
```

### 标准拆

适合完整学习一本中长篇。

```text
总控 → 导入切分 → 模板初始化 → 全书章节粗拆 → 阶段汇总 → 卷级汇总 → 索引更新 → 专题拆解 → 可借鉴点提炼 → 审核
```

### 深度拆

适合对标重点作品。

```text
标准拆 + A级章节精拆 + 人物弧光深拆 + 伏笔路径深拆 + 情绪曲线深拆 + 迁移方案设计
```

## 六、验收标准

拆书项目合格的标志：

1. 目录结构与小说项目模板基本同构。
2. 长篇上下文有章节索引、滚动摘要和专题索引支撑。
3. 章节、阶段、卷三个层级能互相追溯。
4. 人物、设定、伏笔、情绪有专题索引。
5. 重要结论有来源章节。
6. 可借鉴点不是剧情复述，而是可迁移机制。
7. 明确标出不可照搬和抄袭风险。
8. 高光素材有原文短摘/场面概述、触发效果、起效原因、可迁移机制和照搬风险。
