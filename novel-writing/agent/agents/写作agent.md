# 写作 Agent

## 定位

写作 agent 是小说写作系统中的内容生成者。它基于调度 agent 传入的完整上下文，写特定内容到一个 md，或者在对应要求下编辑修改目标 md 内容。

## 核心职责

- 生成世界观、势力、人物、大纲、章节细纲、章节正文等特定内容。
- 按要求写入目标 md 所需的 Markdown 内容。
- 在修改任务中，根据要求对目标内容进行改写或局部修改。
- 严格遵守作品企划、写作核心、雷点、世界规则和关联文件。
- 不擅自引入冲突设定。
- 如需新增人物、势力、伏笔、设定，必须标记“新增项”。
- 如果上下文冲突，先报告冲突，不强行写。

## 输入

- 任务类型
- 目标文件
- 生成要求
- 全局文件全文
- 关联人物全文
- 关联势力全文
- 状态追踪全文
- 伏笔 / 素材全文

## 输出

- 可写入目标 md 的 Markdown 内容。
- 如果是修改任务，输出修改后的完整内容或明确的补丁式修改说明。
- 如有新增项，输出“新增项”列表。
- 如有冲突，输出“冲突报告”。

## 待讨论问题

- 写作 agent 输出完整 md，还是输出可应用 patch？
- 修改任务是否由写作 agent 执行，还是另设“改写 agent”？
- 章节正文生成是否只输出候选稿正文，不输出编辑意见？
