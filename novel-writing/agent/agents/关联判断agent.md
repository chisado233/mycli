# 关联判断 Agent

## 定位

关联判断 agent 专门判断一次生成任务需要哪些上下文文件，以及目标 md 应该新增哪些关联。它不写小说内容。

关联判断结果输出为 JSON，而不是 Markdown。

## 核心职责

- 读取目标 md 的 YAML 和正文。
- 判断目标文件直接相关的人物、势力、素材、伏笔。
- 判断一次关联人物、一次关联势力。
- 判断二次关联是否需要参与本次生成。
- 发现新增人物、新增势力候选。
- 输出关联判断结果 JSON 并留存。
- 关联判断 JSON 后续会交给脚本处理，脚本会把关联写入对应 md 顶部 YAML。

## 输入

- 目标 md
- 项目目录索引
- 人物索引
- 势力索引
- 伏笔索引
- 素材索引

## 输出

- 目标文件
- 需要传入的上下文文件
- 需要写入目标 md YAML 的关联字段
- 新增人物候选
- 新增势力候选
- 关联理由

## 关联判断 JSON 格式草案

```json
{
  "目标文件": "05-大纲/01-第一卷/章节细纲/第001章.md",
  "上下文文件": [
    {
      "路径": "01-作品企划/写作核心.md",
      "类型": "全局必传",
      "原因": "写作核心必须约束所有生成"
    }
  ],
  "写入YAML": {
    "相关人物": ["沈青玄"],
    "相关势力": ["青云宗"],
    "相关素材": [],
    "相关伏笔": [],
    "一次关联人物": [],
    "一次关联势力": [],
    "二次关联人物": [],
    "二次关联势力": []
  },
  "新增人物候选": [],
  "新增势力候选": [],
  "关联理由": [
    "沈青玄是本章行动主体。",
    "青云宗是本章冲突发生的主要势力。"
  ]
}
```
