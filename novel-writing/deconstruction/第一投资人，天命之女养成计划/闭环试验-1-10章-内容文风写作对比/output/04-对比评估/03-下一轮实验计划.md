# 下一轮实验计划

## 1. 下一轮目标

下一轮不建议全量重跑第1-10章。

建议做“针对性小闭环”，重点验证：

1. 内容拆解 agent 增加“场景展开权重”和“喜剧递进层级”后，是否能让写作稿不再像概述。
2. 文风拆解 agent 增加“吐槽强度等级”和“安全替代笑点库”后，是否能在不提高复制风险的情况下增强原文味道。
3. 还原写作 agent 增加“爽点反馈写足”和“高风险桥段替代策略”后，是否能提高爽点、笑点、人物互动。

## 2. 建议重跑章节

优先重跑4章：

| 章节 | 重跑原因 | 验证重点 |
|---|---|---|
| 第4章 | 当前像履历，世俗传奇笑点不足 | 重复梗递进、群体破防、荒诞风流 |
| 第7章 | 当前较好，可作为机制爽点基准章 | 送礼返还、资源标价、配角震惊 |
| 第9章 | 当前祖孙受害史压缩，笑点不足 | 受害者视角重讲主角爽点、喜剧加码 |
| 第10章后半 | 林茵茵登场被削弱 | 高风险女性登场的安全替代写法 |

可选重跑：

| 章节 | 原因 |
|---|---|
| 第3章 | 首次返还爽点应加强，复活丹消耗情绪可加强 |
| 第2章 | 南宫遥登场冲击力可增强 |

## 3. Prompt 修改计划

### 3.1 内容拆解 agent 修改

新增要求：

```markdown
每章必须额外输出：

1. 场景展开权重：
   - 必须展开：
   - 可压缩：
   - 禁止省略：

2. 爽点四段：
   - 压制：
   - 触发：
   - 主角反馈：
   - 配角反馈：
   - 余波新问题：

3. 喜剧递进层级：
   - 第一层：
   - 第二层：
   - 第三层：
   - 爆点/补刀：

4. 高相似桥段替代建议：
   - 原桥段功能：
   - 风险点：
   - 可替代写法：
```

重点让内容拆解 agent 在第4、9、10章给出“不可压缩”的高光场景。

### 3.2 文风拆解 agent 修改

新增要求：

```markdown
输出章节级风味强度表：

| 章节 | 吐槽强度1-5 | 爽点反馈强度1-5 | 外貌描写强度1-5 | 误会喜剧强度1-5 | 情绪保留强度1-5 |

并为高风险桥段提供：
- 安全替代吐槽句式。
- 安全替代外貌描写方式。
- 重复笑点递进模板。
- 配角破防反馈模板。
```

特别要求：

- 对第4章提供“世俗传奇递进梗替代表达”。
- 对第9章提供“多代受害史递进模板”。
- 对第10章提供“钓系女角色登场不照搬原文的行为化写法”。

### 3.3 还原写作 agent 修改

新增硬性约束：

```markdown
写作时必须遵守：

1. 每章至少一个高光场景充分展开，不得只概述。
2. 每次系统返还后，至少写：
   - 主角第一反应。
   - 主角压住表情或算计。
   - 配角外部反馈。
   - 新资源对下一步的影响。
3. 高风险桥段不得简单删弱，必须保留功能并替换表达。
4. 喜剧递进段至少三层加码。
5. 每章结尾钩子不得平收。
```

增加自检输出：

```markdown
## 本章高光自检
- 本章必须展开的高光：
- 是否展开：
- 爽点反馈是否足够：
- 喜剧递进是否至少三层：
- 相似风险如何规避：
```

## 4. 下一轮具体实验方案

### 实验A：第4章重写

目标：

- 强化世俗传奇的荒诞喜剧。
- 不照搬原文“摘桃子”和固定年龄句式。
- 让三个阶段都有群体破防。

判定标准：

| 维度 | 目标 |
|---|---|
| 内容骨架 | ≥4.5 |
| 文风接近 | ≥4.2 |
| 爽点机制 | ≥4.2 |
| 原文相似风险 | ≤3.8 |
| 额外要求 | 三段世俗传奇不得像简历 |

### 实验B：第7章重写

目标：

- 保持当前结构。
- 增强灵石价值标价。
- 增加张仙表面淡定、内心狂喜。
- 增强玉昀阴谋论尾钩。

判定标准：

| 维度 | 目标 |
|---|---|
| 爽点机制 | ≥4.6 |
| 设定释放 | ≥4.7 |
| 文风接近 | ≥4.3 |
| 原文相似风险 | ≤3.6 |

### 实验C：第9章重写

目标：

- 展开祖孙四代受害史。
- 每代有期待、被张仙截断、荒诞后果。
- 让玄微从不信到动摇有过程。
- 保留年轻玄微反转钩子。

判定标准：

| 维度 | 目标 |
|---|---|
| 人物关系 | ≥4.4 |
| 文风接近 | ≥4.3 |
| 爽点/笑点 | ≥4.4 |
| 原文相似风险 | ≤4.0 |

### 实验D：第10章后半重写

目标：

- 林茵茵登场不复刻原文性化动作链。
- 但必须展示：
  - 矿花地位。
  - 男修竞争。
  - 她表面娇憨。
  - 她内心清醒。
  - 她通过小动作获取资源。

判定标准：

| 维度 | 目标 |
|---|---|
| 新人物钩子 | ≥4.4 |
| 文风接近 | ≥4.1 |
| 原文相似风险 | ≤3.5 |
| 额外要求 | 不得只写“漂亮”，必须写“拿捏行为” |

## 5. 进步判定标准

下一轮相较本轮，若满足以下条件，可认为闭环进步：

1. 第4章、第9章不再像概述，而有高光段展开。
2. 爽点后反馈平均提升到4.4以上。
3. 文风接近度平均提升到4.2以上。
4. 原文相似风险不超过本轮。
5. 高风险桥段能保留功能但换表达。
6. 写作稿仍保持正文形态，而不是变成分析或大纲。

## 6. 风险控制

下一轮需要特别注意：

- 不要为了增强文风而照搬原文高识别句。
- 不要直接复用原文低俗比喻。
- 不要复刻林茵茵原文外貌动作链。
- 不要复刻第4章“摘桃子”重复句式。
- 不要复刻玉昀原文标志性哭诉。
- 可以提高缺德感，但不要让主角变成无底线猥琐人物。

## 7. 推荐执行顺序

1. 修改内容拆解 agent prompt。
2. 修改文风拆解 agent prompt。
3. 只重跑第4、7、9、10章拆解与写作。
4. 再进行小范围评估。
5. 如果第4、9章明显改善，再决定是否重跑1-10全章。
