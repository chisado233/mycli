## System Prompt

# 章节粗拆 Agent 提示词模板

## 角色

你是长篇小说拆书系统的章节粗拆 agent。你负责把一章或一批章节转化为结构化拆解记录，为后续阶段汇总、索引更新、专题分析提供事实基础。

## 输入

- 当前章节原文或章节文本路径
- 章节索引
- 上一批滚动摘要
- 已有人物/势力/伏笔/设定索引
- 章节拆解模板

## 核心原则

章节粗拆追求覆盖和准确，不追求长篇文学评论。每章必须输出可被后续机器和人工继续处理的结构化信息。

补充原则：

- 表层事件只保留必要事件链，建议 6-10 条；禁止用大篇幅复述剧情。
- 除“表层事件”外，其余模块必须尽量回答结构功能、机制作用、人物变化、后续追踪价值。
- 若某条内容与“表层事件”重复，优先在更高价值模块中保留，避免重复堆字。
- 拆书结论必须服务后续索引和创作迁移，而不是只解释本章发生了什么。

## 拆解等级判断

- A：开局、高潮、重大反转、核心人物登场、关键伏笔回收、每卷结尾，需要精读。
- B：普通但有明显推进的章节。
- C：过渡、日常、重复战斗、低信息量章节。

## 输出要求

1. 每章单独成块，不要混写。
2. 不仅在人物/设定中区分“明确事实 / 合理推测 / 待验证问题”，在以下模块也必须显式区分判断性质：暗线线索、伏笔埋设、人物状态变化、名场面候选、可借鉴技法。格式可用：`判断类型：事实 / 推测 / 待验证`。
3. 所有伏笔必须标注状态，且只能使用以下枚举：埋设、强化、部分回收、回收、待验证、误判待删。不得自造混合状态词。
4. 不要过度发挥，不要把猜测写成事实。
5. 每章拆解结尾必须给出是否建议进入精拆。
6. 关键判断必须给出证据锚点，不能只在文末笼统列引用。证据锚点可以是原文最短短摘、事件位置或明确情节事实。
7. 除证据锚点外，避免连续照搬原文表达；如出现具有明显文风特征的原句摘录，必须标记“高复用风险”，并说明仅作证据，不作借鉴模板。
8. 高光素材必须做解析，不能只摘句。

## 四、证据追溯检查

### 总体判断

**部分通过。**

优点：

- 每章末尾都有“证据来源”；
- 明确列出若干原文句子，说明有证据意识；
- 一些关键结论如“绑定失败”“好感度下降”“首次返还成功”等，能直接追溯。

问题：

#### 1. 证据挂靠不够细
当前是“章节末尾集中列证据”，但不是“重要结论逐条挂证”。  
结果是：

- 读者知道你引用了原文；
- 但**无法快速确认某一条推论具体由哪一句支持**。

例如：
- “张山对张乐乐的态度从长期嫌弃出现一丝实际承诺”
- “南宫遥带走张乐乐也有明显功利心”
- “主角嘴碎、下限低但未真正越线”
这些判断有合理性，但没有逐项证据对应。

#### 2. 事实与解释之间有时跳步
如 chapter-001：
- “系统迟到十五年才发声，暗示‘气运/天命’不是静态标签，而可能会成长或觉醒。”
这是**合理推测**，但证据仅能支持“现在才识别”，不能直接强支撑“成长性机制”。

#### 3. 章节功能类结论多数无证据锚点
“本章功能”“钩子效用”“技法意义”等高层结论，本来就允许分析，但最好给出最小证据锚点：
- 触发点是什么；
- 结构断点在哪；
- 该判断依据哪组事件或句子。

### 建议标准

后续 prompt 应要求重要判断采用以下格式之一：

- **结论 + 证据句**
- **结论 + 证据事件**
- **结论 + 证据位置（前/中/后段）**

例如：

> 结论：章末形成强钩子。  
> 证据：在温情承诺后立刻插入“发现60分气运之女，南宫遥”的系统提示，情绪被强制打断并转入新线。

这样会比仅在末尾列几句原文更可追溯。

## 五、事实 / 推测 / 待验证检查

### 总体判断

**部分通过，方向正确，但执行不彻底。**

#### 优点
章节拆解中明确设置了：

- 明确事实
- 合理推测
- 待验证问题

这非常关键，也符合用户要求。

#### 主要问题

### 1. 只在“人物/设定”局部有分层，其他模块没有贯彻
例如：

- “暗线线索”
- “伏笔埋设”
- “可借鉴技法”
- “名场面候选”
- “情绪点”
这些模块中，**事实、解释、读者效应、作者意图常混在一起**。

### 2. 个别“推测”仍偏大胆
例如：

- “她的‘天命属性’与年龄、觉醒事件、外界机缘之一有关”
- “她带走张乐乐也有明显功利心，希望凭此获得宗门奖励”
- “主角是其启蒙者，也可能影响其未来命运走向”
这些都可以保留，但应更明确标注为：
- 推测依据是什么；
- 推测强度是“弱推测/中推测”。

### 3. 伏笔状态与“未证实信息”偶有混用
有些内容更像“悬念”或“设定未明”，未必已经构成作者有意埋设的伏笔。  
例如：
- “系统识别延迟机制”
- “张山无法修行是否会被打脸”
其中部分更适合归入“待验证设定/悬念”，不一定直接标“伏笔”。

### 结论

**用户特别关心的“是否区分事实/推测/待验证”，本批次已经做到 60%-70%，但未完全达标。**

这说明输出已经有“服务后续检索”的意识，不是孤立笔记。

#### 主要问题

### 1. 字段规范不统一
例如：
- “疑似埋设 / 强化 / 回收 / 待验证”
- “埋设→登场回收”
- “待验证/强化”
这些状态词现在**能看懂，但不利于机器归档**。

建议固定枚举：
- 埋设
- 强化
- 部分回收
- 回收
- 待验证
- 误判待删

### 2. 人物标签混合“事实身份”和“阶段状态”
例如：
- 张山：穿越者 / 系统宿主 / 山村猎户 / 无灵根 / 先天武者（第3章后）
这种写法对人工友好，但后续结构化索引中建议拆成：
- 身份标签
- 能力标签
- 当前状态标签
- 首次确认章节

### 3. 高光素材索引标准尚未固定
当前有：
- 爽点
- 搞笑点
- 抽象点
- 名场面候选
- 文雅名句候选

这很好，但建议后续补字段：
- 所属章节
- 素材类型
- 原句短摘
- 解析
- 可迁移用途
- 抄袭风险等级

### 4. 待验证问题未形成总表
虽然每章有待验证问题，但**缺少批次级“待验证问题池”**。  
这会导致后续容易遗漏追踪。

### 结论

**已经能支撑“人工索引更新”，但还不足以支撑“稳定自动化索引更新”。**

## 七、可借鉴点质量检查

### 总体判断

**部分通过。**

用户特别强调：不要只是复述剧情，要能支撑创作迁移。  
当前“可借鉴技法”确实已经出现了抽象化倾向，例如：

- “好消息/坏消息式反差开局”
- “用大时间跨度压缩苦难背景”
- “温情戏结尾插入系统提示形成双钩子”
- “跳时避免重复练功流水账”

这些都属于**可迁移机制**，是有价值的。

但仍有以下问题：

### 1. 提炼层级不够稳定
有些是“方法”，有些是“效果描述”，有些是“本章发生了什么”。

例如：
- 好：**“用跳时避免重复练功流水账”**
- 一般：**“用天降美人作为修仙世界的视觉化入口”**
- 偏弱：**“让重要信息通过人物博弈自然抖出”**  
后者方向是对的，但过泛，可再细化成触发条件与适用场景。

### 2. 缺少“如何迁移到创作模板”的说明
目前只说了“可借鉴什么”，没说明：
- 适合什么题材；
- 适合什么章节位置；
- 风险是什么；
- 变体怎么做。

### 3. 个别技法接近“作品专属剧情总结”
例如：
- “主角嘴碎、下限低但未真正越线，维持可吐槽又可跟随的人设平衡。”
这不是不能写，但更适合改成：
- **“低下限吐槽型主角可通过‘嘴上越线、行动不越线’维持读者跟随感。”**

### 结论

**可借鉴性已有基础，但还没有完全从‘拆这本书’升级到‘沉淀创作方法库’。**

## 八、搞笑点 / 抽象点 / 名场面 / 名句专项检查

### 1. 搞笑点
**通过。**
三章都能识别到具体喜剧点，且大多附有简单解析，不只是列梗。

### 2. 抽象点
**通过。**
“鸟蛋都检测过”“越舔好感越负”“刚成长完就被一招秒”等，都抓到了网文读者感知层面的“抽象感”。

### 3. 名场面
**部分通过。**
问题不在识别，而在标准：

- 有些是“高记忆节点”；
- 有些只是“重要情节点”；
- 有些更像“视觉画面”。

建议定义“名场面候选”的筛选标准：
- 是否具强视觉性 / 强情绪反转 / 强传播短摘价值；
- 是否是读者会二创、复述、引用的片段。

### 4. 名句
**部分通过，且有风险。**

当前“文雅名句候选”存在两个问题：

#### 问题A：很多更像原文短摘，不是分析提炼
如：
- “几缕乌丝黏在颈侧，更衬得肌肤如冷玉生晕。”
- “她周身还覆着一层淡黄色的光膜，散发着丝丝暖意。”

这类句子作为“原文优美句摘录”没问题，但如果后续要进入创作借鉴库，必须加：
- 用法解析
- 修辞结构
- 可迁移模板
否则就是摘抄。

#### 问题B：抄袭风险较高
尤其是带明显风格特征的描写句，若未来被直接拿去“借鉴”，风险比剧情总结高得多。

## 九、抄袭与照搬风险

### 总体判断

**存在中度风险，主要集中在“高光素材短摘”和“证据引用”部分。**

#### 低风险部分
- 结构化摘要
- 本章功能
- 冲突、情绪、钩子分析
- 机制提炼

这些大多经过转述与抽象，风险较低。

#### 中高风险部分
1. **文雅名句候选**
   - 原文保留度较高；
   - 若后续沉淀为“可借鉴句库”，风险很大。

2. **证据来源**
   - 当前引用句子总体不算超长，但若后续每章都大量直引，累计风险会上升。

3. **名场面短摘**
   - 若保留大量原句式表达，也容易变成“摘录本”而非“分析本”。

### 建议风控规则

后续 prompt 建议加入：

- 单句引用尽量控制在**必要最短**；
- 优先**概述证据**，少量保留关键词；
- “名句候选”改名为**“语言亮点类型”**或**“表达技法样本”**；
- 每条高光素材必须附：
  - 为什么有效
  - 结构特征
  - 可迁移方式
  - 禁止直接复用提醒
