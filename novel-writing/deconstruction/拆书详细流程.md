# 拆书详细流程

## 目标

把一本小说，尤其是几十万到几百万字的长篇小说，拆成一个可复用、可审核、可追溯、可迁移到创作项目的“对标作品拆解项目”。

拆书成果的目录结构应尽量与：

```text
D:\agent_workspace\capability-library\mycli\novel-writing\project\templates\小说项目模板
```

保持同构，方便后续把拆书结论转化到自己的小说项目模板中。

## 核心原则

1. **不依赖 LLM 长上下文记忆**：长篇小说必须使用章节切分、滚动摘要和专题索引。
2. **不做纯剧情复述**：每个结论都要回答“原作做法、起效原因、可迁移机制、照搬风险”。
3. **分层拆解**：章节 → 阶段 → 卷 → 全书 → 专题 → 可借鉴点。
4. **边生成边审核**：每次 agent 输出都要检查准确性、完整性、证据追溯和可迁移性。
5. **中间过程必须留痕**：请求 JSON、构造提示词、模型输出、审核意见、提示词修订记录都要保存。
6. **摘录要克制**：名场面、名句、搞笑点、抽象点、爽点可以摘原文短句，但必须短摘并附解析，不得大段复制。

## Agent 提示词位置

拆书相关 agent 提示词统一放在：

```text
D:\agent_workspace\capability-library\mycli\novel-writing\agent\prompts\拆书
```

当前核心 agent 包括：

| Agent | 用途 |
|---|---|
| `00-拆书总控agent.md` | 制定拆解计划、批次计划、调用清单 |
| `01-原文导入切分agent.md` | 整理章节、生成章节索引 |
| `02-拆书模板初始化agent.md` | 建立与小说项目模板同构的拆书目录 |
| `03-章节粗拆agent.md` | 分章事实抽取、结构标记、精读建议 |
| `04-章节精拆agent.md` | A 级重点章节技法精读 |
| `05-阶段汇总agent.md` | 将一批章节压缩为阶段总结 |
| `06-卷级汇总agent.md` | 将多个阶段压缩为卷纲式拆解 |
| `07-索引更新agent.md` | 更新人物、势力、伏笔、设定、情绪索引 |
| `08-角色拆解agent.md` | 拆人物设计、人物弧光、角色高光 |
| `09-世界观设定拆解agent.md` | 拆世界观、力量体系、奇观与异常感 |
| `10-明线暗线伏笔拆解agent.md` | 拆明线、暗线、谜团、伏笔路径 |
| `11-作品企划反推agent.md` | 反推类型、卖点、受众、情绪承诺 |
| `12-可借鉴点提炼agent.md` | 输出可迁移机制和可借鉴点总表 |
| `13-审核评估agent.md` | 审核完整性、证据、风险和返工点 |
| `15-高光素材摘录agent.md` | 提取搞笑点、抽象点、名场面、名句、爽点 |

## JSON agent 调用方式

使用 `mycli novel-writing agent run <request.json>`。

请求 JSON 基本格式：

```json
{
  "agent": "拆书-章节粗拆agent",
  "task_type": "章节粗拆",
  "model": "MoreCode/gpt-5.4",
  "max_tokens": 8192,
  "temperature": 0.2,
  "system_prompt": { "path": "D:\\...\\03-章节粗拆agent.md" },
  "user_prompt": "请拆解第001-003章。",
  "project": "D:\\...\\某书拆解项目",
  "target": "07-章节拆解\\chapters\\batch-001-003.md",
  "output_dir": "D:\\...\\11-调度与拆解\\agent-runs",
  "context_files": [
    { "path": "D:\\...\\章节\\001-xxx.md", "label": "第001章" },
    { "path": "D:\\...\\章节\\002-xxx.md", "label": "第002章" }
  ]
}
```

runner 会：

1. 读取 `system_prompt`。
2. 读取 `context_files` 全文。
3. 生成最终提示词。
4. 调用 `mycli agent-cli llm-call`。
5. 保存输出到 `output_dir`。
6. 如果 JSON 中有 `target`，把输出复制到拆书项目目标文件。

## 完整执行流程

### 0. 建立拆书项目

1. 从 `templates/拆书成果模板` 复制一份到具体书籍目录。
2. 填写：
   - `00-拆解总览/作品基本信息.md`
   - `10-拆解记录/拆解进度.md`
   - `10-拆解记录/模型分析记录.md`
3. 记录原文来源：
   - `12-长篇上下文管理/原文切分规则.md`
   - `12-长篇上下文管理/章节索引.md`

### 1. 原文导入与章节索引

目标：确认章节数、标题、文件路径、字数、异常项。

产物：

```text
12-长篇上下文管理/章节索引.md
12-长篇上下文管理/原文切分规则.md
10-拆解记录/阅读记录.md
```

验收：章节编号稳定，路径可追溯，异常章节被标记。

### 2. 首批章节粗拆

建议先拆前 3-10 章，不直接全书跑。

目标：验证提示词是否能正确抽取：

- 一句话概括
- 表层事件
- 本章功能
- 明线/暗线
- 冲突
- 情绪点
- 爽点/虐点/燃点
- 伏笔
- 人物状态变化
- 章节钩子
- 可借鉴技法
- 是否建议精读

产物：

```text
07-章节拆解/chapters/batch-001-003.md
```

验收：不是剧情流水账；关键结论有章节依据；能看出哪些章节应精读。

### 3. 索引更新

基于粗拆结果更新：

```text
12-长篇上下文管理/人物索引.md
12-长篇上下文管理/势力索引.md
12-长篇上下文管理/伏笔索引.md
12-长篇上下文管理/设定索引.md
12-长篇上下文管理/情绪索引.md
12-长篇上下文管理/章节钩子索引.md
12-长篇上下文管理/待验证问题.md
```

验收：索引不求完整，但必须能支撑下一批上下文。

### 4. 阶段汇总

每 10-30 章做一个阶段总结。试拆阶段可以先用第 1-3 章做一个“开局阶段小结”。

产物：

```text
05-大纲/卷模板/阶段细纲/阶段001-开局.md
12-长篇上下文管理/滚动摘要.md
```

验收：能替代这几章进入后续上下文。

### 5. 章节精拆与高光素材摘录

对 A 级章节进行精拆。开局章、系统启动章、第一次返还章、重大反转章优先。

产物：

```text
07-章节拆解/开头章节拆解.md
09-素材引用/名场面与名句摘录.md
09-素材引用/搞笑点与抽象点.md
09-素材引用/爽点摘录与解析.md
12-长篇上下文管理/名场面索引.md
12-长篇上下文管理/名句索引.md
12-长篇上下文管理/梗与抽象点索引.md
```

验收：摘录必须是短摘；每条必须有触发效果、起效原因、可迁移机制和照搬风险。

### 6. 作品企划反推

基于前几章和阶段总结，先做初版企划反推。

产物：

```text
01-作品企划/作品企划总表.md
01-作品企划/类型定位.md
01-作品企划/卖点.md
01-作品企划/情绪承诺.md
```

验收：要能说清楚这本书卖什么、读者为什么追、情绪承诺是什么。

### 7. 可借鉴点提炼

初版只从前几章提炼，不假装全书结论。

产物：

```text
00-拆解总览/可借鉴点总表.md
00-拆解总览/核心结论.md
```

验收：每条都要写清：原作做法、起效原因、可迁移机制、风险、映射到我方模板的位置。

### 8. 审核评估

审核 agent 检查：

- 是否跑题
- 是否过度复述
- 是否缺证据
- 是否缺高光素材
- 是否没有可迁移性
- 是否目录写入位置不对
- 是否存在大段原文复制风险

产物：

```text
08-审核与评估/拆解完整性检查.md
08-审核与评估/可借鉴性评估.md
08-审核与评估/复盘记录.md
```

### 9. 提示词精修

如果审核发现问题，必须回到：

```text
D:\agent_workspace\capability-library\mycli\novel-writing\agent\prompts\拆书
```

修改对应 agent 提示词，并在项目中记录：

```text
10-拆解记录/提示词修订记录.md
```

记录内容：

- 问题是什么
- 影响哪个 agent
- 修改了哪个 prompt 文件
- 修改后预期改善什么
- 是否复验通过

## 中间过程留痕规范

每次 agent 调用都必须保存：

```text
11-调度与拆解/requests/*.json
11-调度与拆解/agent-runs/<timestamp>/提示词.md
11-调度与拆解/agent-runs/<timestamp>/输出.md
11-调度与拆解/agent-runs/<timestamp>/run.json
```

人工或 assistant 操作记录写入：

```text
10-拆解记录/拆解进度.md
10-拆解记录/模型分析记录.md
10-拆解记录/人工判断记录.md
10-拆解记录/提示词修订记录.md
```

## 试拆验收标准

首批试拆合格，应满足：

1. 拆书项目目录完整。
2. 前 3 章或前 5 章已有章节拆解。
3. 至少有一份阶段小结或滚动摘要。
4. 至少有初版人物索引、伏笔索引、设定索引。
5. 至少有高光素材摘录与解析。
6. 至少有一份审核报告。
7. 如果提示词不满足预期，已记录并修订。
