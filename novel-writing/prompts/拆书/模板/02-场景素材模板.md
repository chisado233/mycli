# 场景素材模板

## 适用范围

用于收集可直接转化为小说桥段的场景素材，包括：

- 战斗场景
- 圆房场景
- 亲密场景
- 情绪推进
- 日常互动
- 氛围描写
- 对话场景
- 告白、分别、重逢
- 修炼、渡劫、拍卖、交易、逃亡、探索等

本模板继承并扩展：

```text
D:\agent_workspace\capability-library\mycli\novel-writing\material-library\场景\README.md
```

原 README 的基础要求必须完整保留：场景素材用于记录可直接转化为小说桥段的素材，包括战斗、日常互动、亲密关系、情绪推进、氛围描写、对话、探索、追逐、灾难等。

圆房/做爱/亲密场景只分析叙事功能、关系推进、情绪张力、场面结构和描写策略，不做露骨扩写。

## 最低质量线

一条场景素材必须包含：

- 场景目标。
- 出场人物。
- 场景环境。
- 冲突推进。
- 情绪变化。
- 高光片段或原文摘录。
- 可复用变体。
- 注意事项/禁止照搬点。

额外要求：场景素材不能只是“发生了什么”，而要写出“如何写”。一个合格场景素材应让后续 AI 能直接扩写出类似功能的新场景。

不能只写：

```text
这里是战斗场景，主角很强，敌人被打败。
```

应写成：

```text
敌人先以境界压迫制造生死危机；主角没有临场突破，而是连续砸出超规格资源。每一次出手都让敌人的判断失效：他先以为是一张灵符，再发现是一沓；先以为是一柄法器，再发现主角把上品法器当消耗品自爆。爽点来自“敌人越识货越崩溃”。
```

## 场景分类细化

### 战斗场景

必须记录：战斗目标、双方信息差、战斗节奏、战斗资源、见证者反应、战斗后收益。

### 圆房/亲密场景

只分析叙事功能，不做露骨扩写。必须记录：关系前置、情绪基础、张力来源、场面结构、叙事功能、风险。

### 情绪推进场景

必须记录：初始情绪、触发事件、情绪转折点、未说出口的话、行动如何替代告白/道歉/承诺、情绪落点和后续钩子。

### 日常互动场景

必须记录：日常动作、关系推进、性格暴露、轻松段落如何服务后续冲突或爽点。

### 氛围描写场景

必须记录：场景意象、情绪颜色、视角距离、描写功能。

### 对话场景

必须记录：对话目标、台词节奏、潜台词、对话后的关系变化。

## 拆书提取步骤

1. 判断这个场景是否局部闭合：是否有目标、冲突、变化和结果。
2. 判断是否有复用价值：能否迁移到其他题材或其他人物关系。
3. 提取大段原文证据，保留动作链、对话链、心理链。
4. 抽象场景结构，写出可复用变体。
5. 标记禁止照搬点。

## 文件模板

```markdown
---
name: 场景名称
description: 一句话说明这个场景的用途、冲突和情绪价值。
tag:
  - 场景
  - 具体类型
  - 情绪或功能
status: 成品候选
merge_policy: 可直接入库
source: 原书名
source_range: 第XXX章
---
```
