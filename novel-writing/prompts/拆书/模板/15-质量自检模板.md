# 质量自检模板

质量自检不是形式表格，而是拆书 agent 对自己产物的最低验收。若发现不合格，应在自检中明确指出，并尽可能补救，不要假装通过。

## 批次覆盖检查

```markdown
| 章节 | 是否输出剧情拆解 | 是否产出高价值素材 | 备注 |
|---|---|---|---|
```

## 素材质量检查

- 是否有大段原文证据？
- 是否有写法解析？
- 是否有可复用变体？
- 是否有禁止照搬点？
- 是否避免空泛标签？
- 是否区分成品候选、阶段素材、动态追踪？
- 是否每个素材都有 YAML front matter？
- 是否每个素材都有 `source` 与 `source_range`？
- 是否每个素材至少包含一级分类 tag？
- 是否使用了素材库推荐 tag，而不是自造难检索 tag？
- 是否有 `核心内容 / 原文证据 / 写法解析 / 使用场景 / 可复用变体 / 禁止照搬点` 等等价段落？
- 是否保留了具体动作、对话语气、心理暗流、场面节奏？
- 是否避免“很强、很爽、很暧昧、很有氛围”等空泛词？

## 分类专项检查

### 场景

- 是否写清场景目标、出场人物、环境、冲突推进、情绪变化？
- 场景是否局部闭合？
- 是否能迁移到其他人物/题材？

### 爽点

- 是否写清压制、触发、反馈、余波？
- 是否有见证者反应？
- 是否标记爽点强度？

### 伏笔

- 是否写清埋设位置、表层解释、真实含义、发展节点、回收方式？
- 是否标记状态：未埋设/已埋设/发展中/已回收/待强化等？
- 是否评估“妙处”？

### 人物设定

- 是否有性格具体化？
- 是否有关系型人格？
- 是否有语言、思维、行为示例？
- 是否记录矛盾与裂缝？

### 情感线推进

- 是否写清关系起点、推进阶段、关键事件、情绪变化、阻碍与误会、关系确认方式？
- 是否记录主动权变化？
- 是否标记不可逆节点？

### 力量体系

- 是否写清体系定位、核心资源、等级或境界、晋升条件、战斗表现、限制与代价、克制关系？
- 个人力量体系是否写清当前能力、底牌、短板、成长方向？

### 世界设定

- 是否写清世界类型、地理结构、历史背景、主要势力、基础规则、社会结构、资源与经济、可展开冲突？
- 是否说明该设定如何服务剧情？

### 完整故事概要

- 是否有一句话卖点、主角与目标、世界背景、核心矛盾、主要阶段、关键反转、情感线、结局方向？
- 是否避免流水账？

### 文学风格收集

- 是否说明风格机制？
- 是否有原创可模仿示例？
- 是否说明适用边界？

### 灵活记录

- 是否确实无法归类？
- 是否有可能用途和后续归档建议？
- 是否没有偷懒收纳本该归入其他分类的内容？

## 风险检查

- 是否有大段原文复制但无解析？
- 是否有高识别桥段未标注禁止照搬？
- 是否有把阶段素材误判为最终设定？
- 是否有伏笔状态自造词？
- 是否有目标范围外章节被错误输出？
- 是否遗漏目标范围内章节？
- 是否把 20 章粗扫当成高质量素材拆解？
- 是否因为模型输出不足而只覆盖前几章？
- 是否出现“节选示例”“略”等占位词？
- 是否输出空目录或空模板？
- 是否文件路径不符合多文件输出约定？

## 结论分级

```text
通过：目标章节完整覆盖；高价值素材有证据、有解析、有复用变体；分类与 YAML 合格。
部分通过：章节基本覆盖，但部分素材空泛或分类不完整；需返工指定文件。
不通过：章节覆盖缺失、只有流水账、没有素材解析、没有多文件拆分或大量空模板。
```

## 返工建议模板

如果不是“通过”，必须写：

```markdown
## 返工建议

- 需要补的章节：
- 需要补的分类：
- 需要重写的文件：
- 最严重的问题：
- 建议下一轮 prompt 强化点：
```

## 结论

```text
通过 / 部分通过 / 不通过
```
