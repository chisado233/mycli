# 剧情内容拆解模板

## 定位

剧情内容拆解用于理解原文如何推进故事。它不是可复用素材本身，但为素材归档、阶段汇总、还原写作和最终统一提供结构依据。

剧情内容拆解必须服务三个后续用途：

1. 帮助调度 agent 判断本章有哪些素材值得进入素材库。
2. 帮助阶段汇总 agent 追踪人物、情感、力量、世界、伏笔的变化。
3. 帮助还原写作 agent 抓住章节节奏、冲突链和情绪落点。

因此，剧情拆解不能只写“本章发生了什么”，必须写清“为什么这样安排”“信息如何释放”“读者期待如何被管理”。

## 每章必须拆解

- 本章一句话。
- 表层事件。
- 故事逻辑链：目标 → 阻碍 → 行动 → 结果 → 新问题。
- 场景顺序。
- 人物即时目标。
- 信息释放。
- 对话功能。
- 爽点触发链。
- 伏笔新增/强化/回收。
- 结尾钩子。
- 写作还原输入包。

## 章内拆解细则

### 本章一句话

一句话必须包含：主语、目标、关键行动、结果或新问题。不要写成泛泛总结。

### 表层事件

按读者阅读顺序复述，但只保留推动故事的信息。不要省略关键因果。

### 故事逻辑链

必须填出：目标 → 阻碍 → 行动 → 结果 → 新问题。

如果本章有多个场景，应分别写出多条逻辑链。

### 场景顺序

记录每个场景的入场方式、转场方式和功能。功能可包括：引出冲突、展示实力、推进关系、埋设伏笔、释放爽点、缓和节奏、制造悬念。

### 人物即时目标

必须区分表面目标和真实目标。若人物误解了局势，要写出“错位理解”。

### 信息释放

记录本章新出现的信息：系统规则、人物身份、关系变化、世界设定、伏笔线索、资源数量、敌我实力差。

### 对话功能

对话不能只摘台词，要说明对话在做什么：试探、威胁、撒娇、解释规则、反讽、转移注意、建立关系、制造误会。

### 爽点触发链

按“压制 → 触发 → 反馈 → 余波”写清楚。没有爽点也要说明“无明显爽点”。

### 伏笔变化

标记新增、强化、回收、误导、待验证。必须写明证据。

### 结尾钩子

判断钩子类型：危险钩子、奖励钩子、关系钩子、身份钩子、地图钩子、反派钩子、系统钩子。

## 批次级附加要求

每 5 章批次除逐章拆解外，还应输出：

- 批次主线推进。
- 批次人物变化。
- 批次情感线变化。
- 批次力量体系变化。
- 批次世界设定变化。
- 批次伏笔清单。
- 批次高价值素材候选索引。

## 低质量剧情拆解判定

以下不合格：

- 只复述剧情，没有因果链。
- 只写主角行为，没有阻碍和结果。
- 忽略配角目标。
- 忽略信息释放。
- 忽略结尾钩子。
- 把素材解析混同为剧情流水账。
- 目标章节未覆盖完整。
