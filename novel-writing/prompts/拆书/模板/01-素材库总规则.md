# 素材库总规则

## 定位

素材库不是设定条目仓库，而是供后续 AI 写作者调用的文学素材库。

本规则继承并扩展以下规范，只增不减：

```text
D:\agent_workspace\capability-library\mycli\novel-writing\material-library\README.md
D:\agent_workspace\capability-library\mycli\novel-writing\material-library\TAG-GUIDE.md
```

也就是说：拆书 prompt 可以在这些规范基础上增加更严格的拆解、摘录、解析和复用要求，但不能删除或弱化原有要求。原素材库 README 中关于“文学性优先于资料性”“具体优先于概括”“可写性优先于炫设定”“冲突优先于静态信息”“语言要有质感”“来源必须可追溯”的要求，在所有拆书产物中都强制生效。

拆书时，每一份素材都要让另一个 AI 在阅读后能够直接感知：

- 人物如何行动。
- 场面如何展开。
- 情绪如何变化。
- 冲突如何推进。
- 爽点如何释放。
- 语言和节奏如何被模仿。

## 素材质量要求

- **文学性优先于资料性**：素材可以结构化，但不能只剩设定表。
- **具体优先于概括**：不要只写“很强”“很爽”“很暧昧”，必须写具体动作、对话、心理、旁观者反应。
- **可写性优先于炫设定**：任何设定都要说明能放进什么场景、推动什么冲突。
- **冲突优先于静态信息**：素材要暗含矛盾，如欲望与恐惧、亲近与防备、规则与例外。
- **高质量优先于数量**：宁缺毋滥。没有复用价值的低质量条目不要硬凑。
- **证据与解析并重**：高价值素材必须有原文证据、大段摘录或明确场景证据，并有写法解析。
- **来源必须可追溯**：YAML 中必须填写 `source`；拆书素材应写作品名，必要时补 `source_range` 到章节范围。
- **语言要有质感**：即使是结构化条目，也要保留画面、气味、动作、语气、心理暗流与叙事节奏。
- **只增不减**：如果分类 README 已规定 YAML、正文建议、推荐 tag，本 prompt 只能补充更细标准，不得省略基础字段。

## 文件命名规范

正式素材文件推荐命名：

```text
序号+名称.md
```

要求：

- 序号使用三位数字起步，如 `001`、`002`、`003`。
- 序号和名称之间使用半角加号 `+`。
- 名称应简短、可读、概括素材核心。
- 同一分类目录内序号尽量连续；批次拆解时可先按批次局部编号，阶段整理时再重排。
- 文件名避免过长句子、特殊符号和难以检索的泛称。

## YAML 元数据强制规范

每个素材文件顶部必须包含 YAML front matter。拆书产物至少包含：

```yaml
---
name: 素材名称
description: 一句话说明这个素材的用途、内容或价值。
source: 作品名 / 作者名《作品名》 / 未知
source_range: 第XXX章-第YYY章
tag:
  - 一级分类
  - 具体类型
  - 功能或情绪
status: 成品候选 | 阶段素材 | 动态追踪
merge_policy: 可直接入库 | 阶段汇总 | 最终统一
---
```

字段要求：

- `name`：应与文件名中的名称基本一致。
- `description`：不能空泛，必须说明核心价值。
- `source`：拆书素材必须写原作品名；不确定作者时至少写作品名。
- `source_range`：必须写章节范围，便于回看。
- `tag`：至少包含一个一级分类 tag，并尽量加入分类专用 tag、情绪 tag、功能 tag。
- `status`：用于区分一次可入库素材、阶段性素材、长线追踪素材。
- `merge_policy`：用于告知后续调度 agent 如何处理该素材。

## 每条素材应回答

- 这个素材适合放进什么故事段落？
- 它制造什么情绪？
- 它推动什么：人物关系、主线冲突、伏笔、世界认知、爽点释放，还是性格揭示？
- 它最有价值的细节是什么？
- 如果扩写它，应抓住什么语气、节奏、意象和冲突结构？

## 素材状态标记

建议每个正式素材 YAML 中加入：

```yaml
status: 成品候选 | 阶段素材 | 动态追踪
merge_policy: 可直接入库 | 阶段汇总 | 最终统一
source: 原书名
source_range: 第XXX章-第YYY章
```

含义：

- `成品候选`：单个场景、爽点、桥段、规则，局部闭合，可直接进入精选候选。
- `阶段素材`：人物阶段表现、情感线节点、个人力量体系阶段、局部世界设定，后续要统一。
- `动态追踪`：伏笔、长线设定、可能被后文推翻的判断。

## 原文摘录原则

- 拆书过程允许保留较详细原文摘录，用于私人学习、写法分析、素材定位。
- 高价值素材应保留可回看现场的大段摘录。
- 摘录后必须解析，不能只贴原文。
- 正式迁移到新作品时，不能照搬原句、桥段细节和高识别表达。

## 素材正文最低结构

不同分类可以按各自模板扩展，但至少应包含以下功能段落：

```markdown
## 核心内容

## 原文证据 / 场景证据

## 写法解析

## 使用场景

## 可复用变体

## 注意事项 / 禁止照搬点
```

其中：

- `核心内容`：说明素材是什么。
- `原文证据 / 场景证据`：保留可回看的现场信息，高价值素材要大段摘录。
- `写法解析`：说明原文为什么有效，不能只贴原文。
- `使用场景`：说明后续 AI 能在什么情节里调用。
- `可复用变体`：至少给 2-3 个迁移方向。
- `注意事项 / 禁止照搬点`：指出高识别元素、版权风险、不可照搬的桥段细节。

## 一级分类强制覆盖

当前素材库一级分类包括：

```text
场景
爽点
伏笔
人物设定
力量体系
世界设定
完整故事概要
灵活记录
情感线推进
文学风格收集
```

批次拆书时不要求每个批次都硬凑十类，但如果原文中存在高价值内容，就必须按分类输出，不能把所有内容塞进“灵活记录”。

## 通用 Tag 使用要求

除了一级分类 tag，优先补充以下类型 tag：

- 题材 tag：玄幻、仙侠、修真、都市、系统流、权谋等。
- 剧情阶段 tag：开局、前期、中期、高潮、转折点、爆点、铺垫段等。
- 情绪与氛围 tag：高燃、压抑、暧昧、紧张、诡异、宿命感、残酷等。
- 节奏 tag：快节奏、慢热、爆发、连续反转、情绪递进、强冲突等。
- 读者体验 tag：爽文、虐文、甜文、成长流、无敌流、扮猪吃虎、智斗等。
- 通用功能 tag：展示实力、推进关系、埋设伏笔、回收伏笔、制造悬念、制造爽点、塑造人物等。

Tag 必须服务检索，不要堆无意义标签。

## 坏例与好例标准

坏例：

```markdown
主角很强，很冷静，很聪明。这里可以打脸敌人，读者会很爽。
```

问题：抽象、无动作、无语气、无场面、无可复用结构。

好例：

```markdown
他没有立刻反驳，只把碎裂的茶盏扶正，指腹沾着血，却像是在擦去桌上一点无关紧要的灰。等众人的笑声落下，他才抬眼，说：“你们刚才押的是我的命，还是自己的？”

这一桥段适合先抑后扬式打脸。爽点不来自吼叫，而来自克制后的反击；人物性格应呈现为冷静、记仇、善于等对方把话说死。
```

## 面向后续 AI 的写作提示要求

每份高价值素材都应尽量给出：

- 叙事视角：第一人称、第三人称近景、群像旁观、反派视角等。
- 语言风格：冷峻、华丽、克制、口语、古典、诡异、热血、讽刺等。
- 节奏建议：慢慢压迫、快速爆发、先静后动、连续反转、留白收束等。
- 意象建议：雨、雪、灯火、刀痕、香气、尘土、月光、旧物、伤口等。
- 可扩写钩子：一句台词、一个动作、一个误会、一个未说出口的念头。

## 入库与阶段汇总规则

- `成品候选`：单个场景、爽点、桥段、规则、文风技法等，局部闭合，可进入精选候选。
- `阶段素材`：人物阶段表现、情感线节点、个人力量体系阶段、局部世界设定，必须等待阶段/全书统一。
- `动态追踪`：伏笔、长线设定、可能被后文推翻的判断，必须等待后续验证。

批次产物不要直接全部塞进正式素材库。推荐流程：

```text
批次拆书产物 → 阶段汇总筛选 → 全书统一精选 → 人工确认 → 正式入 material-library
```

## 禁止事项

- 禁止只输出设定表而没有文学质感。
- 禁止把没有复用价值的信息硬凑成素材。
- 禁止把“人物很强”“关系变好”“这里很爽”当作有效解析。
- 禁止把阶段素材误写成最终设定。
- 禁止自造不在规范内且无法解释的伏笔状态。
- 禁止大段原文复制后不做解析。
- 禁止把明确可归类内容偷懒放进灵活记录。
