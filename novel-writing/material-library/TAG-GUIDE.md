# 素材图书馆规范与 Tag 指南

本目录用于保存小说创作素材。所有正式素材必须使用 Markdown 文件，并通过 YAML front matter 记录基础元数据，方便后续检索、归档、自动化处理与项目引用。

## 0. 文学性总要求

本素材库面向后续 AI 写作者使用。素材不能只提供“概念正确”的信息，还要提供“可被写成小说”的质感。

写入任何素材时，请优先保留：

- 具体画面，而不是抽象判断。
- 人物动作，而不是单纯形容词。
- 对话语气，而不是剧情说明。
- 情绪转折，而不是静态情绪标签。
- 意象、节奏和留白，而不是纯设定罗列。

坏例子：

```markdown
这里主角很愤怒，然后爆发，打败敌人。
```

好例子：

```markdown
他听完没有立刻拔剑，只低头把袖口的血一点点抹平。等敌人笑到第三声，他才抬眼，声音比雪还冷：“现在轮到我问了。”
```

后续 AI 读取素材后，应能判断：这个素材适合什么语气、什么节奏、什么人物状态，以及应该用哪类细节扩写。

## 1. 文件格式

所有素材文件必须是 `.md` 文件。

推荐命名格式：

```text
序号+名称.md
```

示例：

```text
001+雨夜刺杀.md
002+退婚打脸.md
003+宗门境界体系.md
004+青梅竹马关系破裂.md
```

命名规则：

- 序号使用三位数字起步：`001`、`002`、`003`。
- 序号和名称之间使用半角加号：`+`。
- 名称应简短、可读、能概括素材核心。
- 不建议在文件名中使用过长句子。
- 同一文件夹内序号尽量连续，但允许后续整理时重排。

## 2. YAML 元数据规范

每个素材文件顶部必须包含 YAML front matter。

最小模板：

```yaml
---
name: 素材名称
description: 一句话说明这个素材的用途、内容或价值。
source: 原创 / 作品名 / 作者名《作品名》 / 未知
tag:
  - 大类
  - 具体类型
  - 功能或情绪
---
```

字段说明：

- `name`：素材名称，应与文件名中的名称基本一致。
- `description`：一句话简介，说明素材核心价值。
- `source`：素材出自哪里。原创素材写 `原创`；来自具体作品时写清作品名，建议格式为 `作者名《作品名》`；不确定来源写 `未知`，后续再补。
- `tag`：标签列表，可以有多个。至少包含所属大类 tag。

正文建议结构：

```markdown
## 核心内容

## 使用场景

## 可变体

## 注意事项
```

不同分类可以根据自己的 README 模板扩展正文结构。

## 6. 推荐使用方式

每个素材建议至少包含：

1. 一个一级分类 tag。
2. 一个具体类型 tag。
3. 一个功能、情绪、题材或状态 tag。

示例：

```yaml
---
name: 雨夜反杀
description: 主角在雨夜被伏击，借环境和隐藏实力完成反杀。
tag:
  - 场景
  - 战斗场景
  - 雨夜
  - 反杀
  - 高燃
  - 展示实力
---
```

```yaml
---
name: 退婚当众打脸
description: 主角在退婚羞辱场景中揭露实力，完成群体见证下的反转。
tag:
  - 爽点
  - 打脸
  - 当众打脸
  - 群体见证
  - 先抑后扬
  - 爽快
---
```

```yaml
---
name: 从互相试探到信任建立
description: 两个角色从戒备、试探到愿意托付后背的情感线推进模型。
tag:
  - 情感线推进
  - 试探
  - 信任建立
  - 推进关系
  - 情绪递进
---
```
