# 人物设定素材规范

本目录用于记录角色设定，包括主角、配角、反派、导师、宿敌、家族成员、组织成员等，也可以记录角色性格、动机、成长弧光和关系网络。

人物设定必须具有文学可写性。不要只写身份、能力和履历；要让后续 AI 能写出这个人的声音、眼神、选择方式、亲密关系中的反应，以及他在压力下暴露出的真实自我。

## 人物文学性要求

### 1. 性格必须具体化

人物不能只用标签定义。`冷静`、`温柔`、`疯批`、`隐忍`、`腹黑` 只是入口，不是人物本身。每个主要人物都应说明：

- 他在安全状态下是什么样？
- 他在被羞辱、被误解、失去重要之物时是什么样？
- 他如何表达喜欢、厌恶、恐惧、愧疚和占有欲？
- 他习惯压抑情绪，还是立刻行动？
- 他最不愿承认的欲望是什么？
- 他最容易重复的错误是什么？

推荐写法：

```markdown
他不是没有情绪，而是习惯把情绪延后处理。被冒犯时不会立刻发怒，只会记住对方说话的顺序、在场的人、能被利用的漏洞。真正动手时，他往往已经把退路和代价都算清楚。
```

### 2. 强调人物关系型

人物不是孤立存在的。每个重要人物都应说明他在不同关系中的“关系型人格”，也就是他面对不同对象时会变成什么样。

至少记录：

- **亲密关系中**：依赖、克制、回避、控制、保护、试探，还是献祭？
- **敌对关系中**：羞辱、沉默、交易、戏弄、碾压，还是给对方留余地？
- **权力关系中**：服从、反叛、伪装顺从、操控上位者，还是天然支配？
- **弱者面前**：怜悯、利用、无视、保护，还是投射自己的过去？
- **强者面前**：崇拜、嫉妒、挑战、学习，还是暗中布局？

关系型写法示例：

```markdown
他对陌生人礼貌而疏离，对敌人极有耐心，对亲近之人反而笨拙。越是在意谁，越不愿直接说出关心，只会提前把路铺好，再假装那只是顺手。
```

### 3. 必须包含语言、思维、行为示例

主要人物素材必须提供三类示例，供后续 AI 模仿人物一致性。

#### 语言示例

记录人物说话方式，而不是只写“他说话冷淡”。包括句长、称呼、口头禅、讽刺方式、是否解释、是否命令式表达。

```markdown
## 语言示例

- “你可以继续说。我不打断将死之人的遗言。”
- “不是原谅你，是现在杀你不划算。”
- 说话特点：短句多，少解释，习惯把威胁说得像陈述事实。
```

#### 思维示例

记录人物如何理解世界、如何做判断、如何自我欺骗。

```markdown
## 思维示例

- 他判断一个人是否可信，看的不是誓言，而是对方在利益受损时的第一反应。
- 他习惯先寻找最坏结局，再决定要不要相信眼前的温情。
- 他常把“我想保护她”伪装成“她还有利用价值”。
```

#### 行为示例

记录人物在具体场景中会做什么。行为要能暴露性格。

```markdown
## 行为示例

- 听到坏消息时，先整理袖口，再问死了几个人。
- 安慰人时不会拥抱，只会把热茶推过去，说“喝完再哭”。
- 动怒时反而降低音量，开始称呼对方全名。
```

### 4. 人物要有矛盾与裂缝

一个可写的人物应有自相矛盾之处。不要把人物写成纯标签集合。

可记录：

- 公开形象 vs 私下状态
- 自我认知 vs 他人眼中的他
- 口头原则 vs 真正会破例的人或事
- 想要的东西 vs 不敢承认的东西
- 爱人的方式 vs 伤人的方式

示例：

```markdown
他自称最讨厌麻烦，却总是在她遇险前提前出现；他厌恶承诺，却记得她随口说过的每一个小愿望。
```

## 人物素材最低合格线

如果是主要人物，至少应包含：

- 3 个以上性格关键词，并解释这些关键词如何在行动中体现。
- 2 组以上人物关系型说明，例如“对恋人”“对敌人”“对师长”“对弱者”。
- 3 条语言示例。
- 3 条思维示例。
- 3 条行为示例。
- 至少 1 个核心矛盾或破例点。

如果是次要人物，也至少应有一个鲜明行为习惯或一句可识别的说话方式。
