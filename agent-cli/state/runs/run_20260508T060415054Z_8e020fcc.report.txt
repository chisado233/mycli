现在可用命令：

```powershell
mycli startup --help
mycli startup commands
mycli startup add <id> <command...>
mycli startup remove <id>
mycli startup enable <id>
mycli startup disable <id>
mycli startup install
mycli startup uninstall
mycli startup run
mycli startup status
mycli startup native <args...>
```

设计行为：

- `mycli startup add ...` 会把命令注册到启动注册表，并自动安装/更新 Windows 登录启动计划任务。
- 开机/登录时会运行计划任务：
  - `\mycli\startup\RunRegisteredCommands`
- 计划任务会调用：
  - `mycli startup run`
- `run` 会异步启动所有 `enabled=true` 的注册命令，并把日志写到：
  - `D:\agent_workspace\capability-library\mycli\startup\state\logs\`
- 注册表位置：
  - `D:\agent_workspace\capability-library\mycli\startup\state\startup-commands.json`

示例用法：

```powershell
mycli startup add task-hall-recover mycli task-hall lifecycle-wake --reason startup-recover --cwd D:\agent_workspace
mycli startup commands
mycli startup status
```

已验证：

```powershell
mycli list
mycli startup --help
mycli startup command list
mycli startup commands
mycli startup status
mycli startup run
```

结果正常。当前没有实际注册任何开机启动命令，也没有创建注册表文件；只有在执行 `mycli startup add ...` 或 `install` 时才会真正写状态/装计划任务。
