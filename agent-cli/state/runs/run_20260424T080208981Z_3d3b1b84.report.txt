已补完，并且通过 `mycli project-manager` 暴露出来了。现在它不只是项目注册/查询工具，也可以作为 **agent 团队接手项目时的项目上下文入口**。

## 新增能力

### 1. 动态上下文补全

现在每个项目可以按阶段持续补充上下文：

```text
.agent-project/context/entries/context-0001.json
```

适合记录：

- 需求阶段：用户偏好、范围、边界；
- 可行性阶段：风险、调研结论；
- 架构阶段：架构取舍、依赖、约束；
- 施工阶段：实现注意事项；
- 验证阶段：测试发现、验收标准；
- 运行阶段：运行记录、异常模式；
- 维护阶段：修复准则、维护注意事项。

命令：

```powershell
mycli project-manager context-add <project> --title <title> --phase <phase> --summary <text>
mycli project-manager context-list <project> [--phase <phase>]
mycli project-manager context-get <project> <context-id>
```

示例：

```powershell
mycli project-manager context-add project-manager `
  --title "Agent takeover context" `
  --phase delivery `
  --type handoff `
  --priority high `
  --summary "Agent teams should start with current, agent-guide, task-list, next-list, and status before modifying this project." `
  --file README.md `
  --file SKILL.md `
  --tag handoff `
  --tag guide
```

---

### 2. 当前项目快照

新增：

```powershell
mycli project-manager current <project>
```

它会输出：

- 当前生命周期；
- 当前阶段；
- 当前活动状态；
- health / delivery / attention；
- current summary；
- 当前 task；
- 当前 next-action。

支持 JSON：

```powershell
mycli project-manager current <project> --json
```

---

### 3. agent 团队接手指南

新增：

```powershell
mycli project-manager agent-guide <project>
mycli project-manager agent-guide <project> --phase validation
```

它会输出一份 agent 接手项目的说明，包括：

- 项目身份；
- 当前状态；
- 当前 focus；
- 项目管理准则；
- 维护准则；
- open tasks；
- open next-actions；
- 指定阶段的动态上下文。

这就是后续 agent 团队接手项目时的入口命令。

建议接手项目第一步执行：

```powershell
mycli project-manager current <project>
mycli project-manager agent-guide <project>
mycli project-manager status <project>
mycli project-manager task-list <project>
mycli project-manager next-list <project>
```

---

### 4. 维护准则入口

新增：

```powershell
mycli project-manager maintenance-guide <project>
```

它会输出：

- agent handoff guide；
- maintenance phase 相关动态上下文；
- operations / maintenance snapshot；
- ongoing 项目的运行信息；
- finite 项目则提示 operations not configured。
