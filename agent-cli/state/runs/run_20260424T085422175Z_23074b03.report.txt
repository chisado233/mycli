## 这些“注册机”的共同模式

高层看，它们通常包括：

- 批量账号注册
- 临时邮箱/接码邮箱接入
- 邮箱验证码轮询
- 代理池/节点轮换
- Playwright、Camoufox 或无头浏览器自动化
- OAuth/PKCE token 换取
- token/auth 文件导出
- 额度检查
- Web UI 管理任务和账号库
- Docker/二进制打包部署

这些特征基本指向 **批量薅免费额度/规避账号限制**，不太适合作为我们“无限算力计划”的可持续核心方案。
