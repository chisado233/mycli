### 2. “active work” 的实际判定信号
从脚本和状态文件看，关键不是泛泛地看大厅里有没有记录，而是看这些字段/状态：

#### 任务层
主状态在：
- `task-hall/state/listings.json`
- `task-hall/state/tasks/<task-id>/meta.json`

主要信号：
- `status == "claimed"`：有人正在处理
- `claimed_by != null`：明确已被某 agent 占用
- `status == "listed"`：可领取，但未被处理
- `status == "draft"`：存在草稿，但未进入可领取大厅
- `status in ("done","cancelled","archived")`：不算当前活跃执行工作

脚本依据：
- `Set-TaskStatus` 会更新状态并同步 `listings.json`
- `Invoke-Claim` 只允许 `listed` → `claimed`
- `Invoke-Tasks` 默认只显示 `listed`，不会默认显示 `claimed`

所以：
- **“task-hall 是否有 active work”不能只看 `tasks` 默认输出**
- 还要结合 `claimed` 状态和 `claimed_by`

### 3. 事件与审计信号
