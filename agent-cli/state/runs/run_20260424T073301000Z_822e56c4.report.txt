已继续优化，并且已经把 `project-manager` 自己作为一个真实项目注册进去了。

## 做了什么

### 1. 优化默认 registry 路径

之前默认 registry 写在源目录：

```text
agent-system-rules/project-manager/registry.json
```

我已经改成优先写入 mycli 包目录：

```text
D:\agent_workspace\capability-library\mycli\project-manager\registry.json
```

如果 mycli 包目录不存在，才回退到源目录。

这样更符合使用直觉：

```text
mycli project-manager query
```

默认查询的是 mycli 侧 registry。
